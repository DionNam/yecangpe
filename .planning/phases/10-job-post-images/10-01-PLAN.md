<task type="auto">
  <name>Task 1: Create database migration for image_url and storage bucket</name>
  <files>supabase/migrations/00006_add_job_images.sql</files>
  <action>
Create migration file with:

1. Add image_url column to job_posts table:
```sql
ALTER TABLE public.job_posts
ADD COLUMN image_url TEXT;
```

2. Create the job-images storage bucket:
```sql
INSERT INTO storage.buckets (id, name, public)
VALUES ('job-images', 'job-images', true)
ON CONFLICT (id) DO NOTHING;
```

3. Create RLS policies for storage.objects:

Policy 1 - Insert (employers and admins can upload):
```sql
CREATE POLICY "Employers and admins can upload job images"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id = 'job-images'
  AND (
    (SELECT public.is_employer())
    OR (SELECT public.is_admin())
  )
);
```

Policy 2 - Select (anyone can view, images are public):
```sql
CREATE POLICY "Anyone can view job images"
ON storage.objects FOR SELECT
TO anon, authenticated
USING (bucket_id = 'job-images');
```

Policy 3 - Update (for replacing images):
```sql
CREATE POLICY "Employers and admins can update job images"
ON storage.objects FOR UPDATE
TO authenticated
USING (bucket_id = 'job-images')
WITH CHECK (
  bucket_id = 'job-images'
  AND (
    (SELECT public.is_employer())
    OR (SELECT public.is_admin())
  )
);
```

Policy 4 - Delete (for removing images):
```sql
CREATE POLICY "Employers and admins can delete job images"
ON storage.objects FOR DELETE
TO authenticated
USING (
  bucket_id = 'job-images'
  AND (
    (SELECT public.is_employer())
    OR (SELECT public.is_admin())
  )
);
```

Use standard migration format matching existing files (00001-00005).

<task type="auto">
  <name>Task 2: Configure Next.js remote image patterns</name>
  <files>apps/web/next.config.ts, apps/admin/next.config.ts</files>
  <action>
Update both Next.js config files to allow loading images from Supabase storage.

For apps/web/next.config.ts, add images configuration:
```typescript
import type { NextConfig } from 'next'
import path from 'path'

const nextConfig: NextConfig = {
  output: 'standalone',
  outputFileTracingRoot: path.join(__dirname, '../../'),
  transpilePackages: ['@repo/supabase', '@repo/lib'],
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '*.supabase.co',
        pathname: '/storage/v1/object/public/**',
      },
    ],
  },
}

export default nextConfig
```

Apply the same images configuration to apps/admin/next.config.ts.

Use wildcard hostname `*.supabase.co` to work with both local (127.0.0.1) and production Supabase URLs.

Note: Local development uses HTTP, but Next.js Image component handles this automatically when using unoptimized images or when the dev server proxies requests.
