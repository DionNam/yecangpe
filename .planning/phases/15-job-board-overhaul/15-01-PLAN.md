  <name>Task 1: Create FTS migration and apply to database</name>
  <files>supabase/migrations/20260207_add_fts_column.sql</files>
  <action>
    Create a SQL migration file that:
    1. Adds a generated `fts` tsvector column to `job_posts` that concatenates `title` and `description` (the `content` column in the codebase maps to `description` in DB? Check actual DB column name - it may be `content`). Use `COALESCE` for null safety:
       ```sql
       ALTER TABLE job_posts
         ADD COLUMN IF NOT EXISTS fts tsvector
         GENERATED ALWAYS AS (
           to_tsvector('english', COALESCE(title, '') || ' ' || COALESCE(content, ''))
         ) STORED;
       ```
       Note: The content column in the codebase is called `content` in the DB. Verify by checking the existing migration files or the job_posts table structure.
    2. Create a GIN index for fast full-text search:
       ```sql
       CREATE INDEX IF NOT EXISTS job_posts_fts_idx ON job_posts USING gin(fts);
       ```

    Apply this migration to the Supabase database using `supabase db push` or execute directly via the Supabase SQL Editor/MCP tool.

    IMPORTANT: Use `COALESCE` to handle NULL values in title/content columns. The generated column must handle cases where either field is NULL.
    IMPORTANT: Use 'english' language config for to_tsvector since job postings contain both English and Korean text. The 'english' config provides stemming for English words while still indexing Korean characters.
  </action>
  <verify>
    Run a test query against the database to confirm:
    1. The `fts` column exists on `job_posts`
    2. The GIN index `job_posts_fts_idx` exists
    3. A sample full-text search works:
       ```sql
       SELECT id, title FROM job_posts WHERE fts @@ websearch_to_tsquery('english', 'test') LIMIT 1;
       ```
  </verify>
  <done>
    FTS column and GIN index exist on job_posts table. Full-text search queries return results correctly.
  </done>
