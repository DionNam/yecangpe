  <name>Task 1: Create filter page data configuration</name>
  <files>apps/web/lib/filter-page-data.ts</files>
  <action>
Create a comprehensive data/config file that provides all filter-specific content for the 5 filter dimensions.

**Filter dimensions to cover:**
1. `by-type` - JOB_TYPES (6 values: full_time, part_time, contract, freelance, internship, temporary)
2. `by-location-type` - Location types (3 values: remote, on_site, hybrid)
3. `by-country` - COUNTRIES (26 values from countries.ts)
4. `by-category` - CATEGORIES (20 values from categories.ts)
5. `by-language-level` - KOREAN_LEVELS (filter out not_specified, keep 5: native, advanced, intermediate, basic, not_required)

**What this file must export:**

1. `LOCATION_TYPES` constant array: `[{ code: 'remote', name: 'Remote', nameKo: '원격' }, { code: 'on_site', name: 'On-site', nameKo: '현장' }, { code: 'hybrid', name: 'Hybrid', nameKo: '하이브리드' }]` - follows same pattern as JOB_TYPES.

2. `type FilterDimension = 'by-type' | 'by-location-type' | 'by-country' | 'by-category' | 'by-language-level'`

3. `getFilterPageTitle(dimension: FilterDimension, code: string): { title: string, titleKo: string }` - Returns bilingual title for any filter value. Examples:
   - `('by-type', 'full_time')` -> `{ title: 'Full-time Jobs', titleKo: '정규직 채용 공고' }`
   - `('by-country', 'KR')` -> `{ title: 'Jobs in South Korea', titleKo: '대한민국 채용 공고' }`
   - `('by-category', 'it_engineering')` -> `{ title: 'IT / Engineering Jobs', titleKo: 'IT/엔지니어링 채용 공고' }`
   - `('by-language-level', 'advanced')` -> `{ title: 'Advanced Korean Level Jobs', titleKo: '한국어 고급 채용 공고' }`

4. `getFilterPageDescription(dimension: FilterDimension, code: string): string` - SEO description for metadata. Should be unique per filter value, ~150-160 chars, include "Korean-speaking" and "HangulJobs". Example: "Browse 정규직 (Full-time) job opportunities for Korean-speaking professionals on HangulJobs. Find verified positions from trusted employers."

5. `getFilterPageFAQs(dimension: FilterDimension, code: string): Array<{ question: string; answer: string }>` - Returns 3-4 FAQs per filter page. FAQs should be genuinely helpful and specific to the filter value. NOT the same generic FAQs on every page. Example for 'full_time':
   - Q: "What full-time benefits are typical for Korean-speaking positions?" A: "[Genuine helpful answer about typical benefits]"
   - Q: "How competitive is the full-time job market for Korean speakers?" A: "[Helpful answer]"
   - Q: "Do full-time positions on HangulJobs require TOPIK certification?" A: "[Helpful answer]"

   For categories, tailor FAQs to the industry. For countries, tailor to the local job market. Create at least a default FAQ set per dimension, and specific FAQs for the top 3-4 values per dimension.

6. `getFilterCrossLinks(dimension: FilterDimension, currentCode: string): Array<{ name: string; nameKo: string; url: string; description: string }>` - Returns cross-links to OTHER filter dimensions (not same-dimension values). Example for by-type/full_time: link to /jobs/by-location-type/remote, /jobs/by-category/it_engineering, /jobs/by-language-level/advanced, etc. Return 6-8 cross-links mixing different dimensions. Use keyword-rich descriptions.

7. `getFilterDBColumn(dimension: FilterDimension): string` - Maps dimension to Supabase column name:
   - 'by-type' -> 'job_type'
   - 'by-location-type' -> 'work_location_type'
   - 'by-country' -> 'work_location_country'
   - 'by-category' -> 'category'
   - 'by-language-level' -> 'korean_level'

8. `getFilterValues(dimension: FilterDimension): Array<{ code: string; name: string; nameKo: string }>` - Returns the valid filter values for a dimension, using existing constants. For countries, map { code, nameEn, name } to { code, name: nameEn, nameKo: name }.

9. `generateFilterPageMetadata(dimension: FilterDimension, code: string, jobCount?: number): Metadata` - Returns Next.js Metadata object with title, description, alternates.canonical, openGraph (title, description, url, type: 'website', locale: 'ko_KR', siteName: 'HangulJobs'), twitter (card: 'summary_large_image', title, description). Import Metadata from 'next'.

Import constants from '@repo/lib' (JOB_TYPES, CATEGORIES, KOREAN_LEVELS, COUNTRIES). Do NOT import from relative paths to the constants.
