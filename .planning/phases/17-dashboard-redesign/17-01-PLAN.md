  <action>
Create `apps/web/app/(main)/dashboard/page.tsx` as a server component with the following logic:

1. Authenticate user via `createClient()` + `supabase.auth.getUser()`. If no user, `redirect('/login')`.

2. Check for employer profile: `supabase.from('employer_profiles').select('*').eq('user_id', user.id).maybeSingle()`. If found:
   - Fetch employer's job posts: `supabase.from('job_posts').select('*').eq('author_id', user.id).order('created_at', { ascending: false })`
   - Fetch like counts for all posts (same pattern as current `/employer/posts/page.tsx`)
   - Render a temporary placeholder: a `<div>` with heading "고용주 대시보드" and text showing `{posts.length}` posts found. Include a Link to `/dashboard/post-job` saying "새 공고 작성". This placeholder will be replaced by the EmployerDashboard component in Plan 17-02.

3. Check for seeker profile: `supabase.from('seeker_profiles').select('*').eq('user_id', user.id).maybeSingle()`. If found:
   - Fetch liked jobs (same pattern as current `/my-page/page.tsx`)
   - Fetch job alerts: `supabase.from('job_alerts').select('*').eq('user_id', user.id)`
   - Render a temporary placeholder: a `<div>` with heading "구직자 대시보드" showing profile nationality and `{likedJobs.length}` liked jobs. This placeholder will be replaced by SeekerDashboard in Plan 17-04.

4. If neither profile found, `redirect('/onboarding')`.

Use `(supabase as any)` pattern for queries that don't match generated types (same pattern used throughout codebase).

  title: '대시보드',
}
