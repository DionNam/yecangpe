**2. Create `apps/web/components/dashboard/job-post-table.tsx`** - Client component ('use client'):

Props: `{ posts: JobPost[], likeCounts: Record<string, number>, onPostDeleted: () => void }`

Key implementation details:
- Split posts into `activePosts` (where `expires_at` is null OR `new Date(expires_at) > new Date()`) and `expiredPosts` (where `expires_at` exists AND `new Date(expires_at) <= new Date()`)
- Show **REAL metrics** (not fake/algorithmic). For each post display:
  - `review_status` badge: 게시중 (published/green), 검토중 (pending/yellow), 반려됨 (rejected/red)
  - Title as link to `/jobs/{slug || id}` (only for published posts)
  - Rejection reason below title for rejected posts
  - `view_count` (real, as-is from DB - number with Eye icon)
  - `apply_click_count` (real, as-is from DB - the field may be 0 or null, show 0 if null)
  - `likeCounts[post.id]` (real like count from likes table)
  - Published date (formatted with `toLocaleDateString('ko-KR')`)
  - Expires date (from `expires_at`, formatted same way)
  - Edit button: Opens `PostEditModal` (reuse existing `@/components/employer/post-edit-modal`)
  - Delete button: Calls `deleteJobPost` server action with confirmation dialog (use `window.confirm('이 공고를 삭제하시겠습니까?')`)

- Active posts section: heading "활성 공고 ({count})" with full table
- Expired posts section: heading "만료된 공고 ({count})" with dimmed styling (opacity-60), only show if expiredPosts.length > 0
- Empty state: "등록한 공고가 없습니다"

Use shadcn/ui Table, Badge, Button components. Import `PostEditModal` from `@/components/employer/post-edit-modal`.

**CRITICAL**: Do NOT use `getDisplayMetrics()` for the employer dashboard. Show raw `post.view_count` and `post.apply_click_count` directly. Fake metrics are only for public-facing pages.
