<task type="auto">
  <name>Task 1: Create job detail page with view tracking</name>
  <files>
    apps/web/app/(main)/jobs/[id]/page.tsx
  </files>
  <action>
    Create job detail page at `apps/web/app/(main)/jobs/[id]/page.tsx`:

    1. Server component with params: { id: string }
    2. Await params (Next.js 15 pattern)
    3. Create Supabase client, get user with getUser()
    4. If no user, redirect to /login (use next/navigation redirect)
    5. Fetch the job post by id:
       - `.select('*')` from job_posts
       - `.eq('id', id)` and `.eq('review_status', 'published')`
       - `.single()`
       - If not found, use notFound() from next/navigation
    6. Call increment_view_count RPC (await supabase.rpc('increment_view_count', { post_id: id }))
       - This is a SECURITY DEFINER function, so it bypasses RLS
       - Call AFTER fetching to ensure post exists
    7. Fetch global_metrics_config to get ramp_days, curve_strength
       - If config not accessible (RLS), use defaults: rampDays: 14, curveStrength: 2.0
    8. Get like count using RPC: supabase.rpc('get_like_count', { post_id: id })
    9. Check if current user liked the post: supabase.rpc('user_liked_post', { post_id: id })
    10. Calculate display metrics using getDisplayMetrics from lib/utils/metrics
    11. Get user role to determine if they can like (seeker only)
        - Query seeker_profiles where user_id = user.id
        - If exists, canLike = true
    12. Pass all data to JobDetail component
  </action>
  <verify>
    - `pnpm build` passes
    - Navigate to /jobs/[valid-id] when logged in - shows job detail
    - Navigate to /jobs/[valid-id] when logged out - redirects to /login
    - Check Supabase dashboard: view_count incremented after page load
  </verify>
  <done>Job detail page fetches job data, increments view count, and calculates display metrics</done>
</task>
