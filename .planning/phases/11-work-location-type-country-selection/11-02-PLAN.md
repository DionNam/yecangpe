  <action>
Update apps/web/lib/validations/job-post.ts following RESEARCH.md "Validation Schema with Conditional Logic" example:

1. Import COUNTRIES from '@repo/lib' (add to existing import with NATIONALITIES)
2. Extract country codes: `const countryCodes = COUNTRIES.map(c => c.code) as [string, ...string[]]`
3. Add to jobPostSchema:
   - work_location_type: z.enum(['remote', 'hybrid', 'on_site'], { message: '근무 형태를 선택해주세요' })
   - work_location_country: z.enum(countryCodes).optional()
4. Add .superRefine((data, ctx) => {...}) after schema object for conditional validation:
   - If work_location_type is 'on_site' and work_location_country is missing, add issue with path ['work_location_country']
   - If work_location_type is NOT 'on_site', set work_location_country to undefined (clear it)
5. Add work_location_type and work_location_country to jobPostUpdateSchema (but NOT in superRefine - update schema doesn't change location)

Follow the exact validation message format from RESEARCH.md: "대면 근무의 경우 근무 국가를 선택해주세요"

DO NOT use useEffect to clear field - Zod's superRefine handles clearing in validation.
