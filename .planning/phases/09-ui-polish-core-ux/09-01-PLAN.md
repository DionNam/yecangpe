<task type="auto">
  <name>Task 2: Create sign out route handler with cache revalidation</name>
  <files>
    apps/web/app/auth/signout/route.ts
  </files>
  <action>
    Create POST route handler following official Supabase pattern from 09-RESEARCH.md.

    Implementation requirements:
    - Check if user is logged in via getUser()
    - Call supabase.auth.signOut() if user exists
    - Use revalidatePath('/', 'layout') to clear server cache (prevents stale auth state)
    - Redirect to '/' (landing page, not /login - allows browsing public job list)
    - Return 302 status

    Pattern source: https://github.com/supabase/supabase/blob/master/examples/user-management/nextjs-user-management/app/auth/signout/route.ts

    Why route handler instead of client-only:
    - Server cache invalidation (revalidatePath) only works in route handlers
    - Ensures middleware re-evaluates auth state on next navigation
    - Proper cookie clearing via Supabase server client

    Use @repo/supabase/server for createClient (matches existing auth patterns).
  </action>
  <verify>

    Should show all three functions present.
  </verify>

    - Handler calls supabase.auth.signOut()
    - Handler calls revalidatePath('/', 'layout')
