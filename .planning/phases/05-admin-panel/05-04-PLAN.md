# 05-04: Metrics Configuration

## Goal

관리자가 조작 조회수와 관심수의 전역 설정(목표 범위, 기간, 커브강도)을 관리할 수 있는 설정 페이지를 구현한다.

## Requirements Covered

- ADMM-01: 관리자는 조작 조회수 전역 설정(목표 범위, 기간, 커브강도)을 할 수 있다
- ADMM-02: 관리자는 조작 관심수 전역 설정(목표 범위, 기간, 커브강도)을 할 수 있다

## Context

05-01에서 관리자 인증과 대시보드 레이아웃 완성됨. global_metrics_config 테이블에 view_target_min/max, like_target_min/max, ramp_days, curve_strength 필드 존재. RLS에서 admin만 읽기/수정 가능. 05-CONTEXT.md에서 사용자가 결정한 설정 UI 방식 반영.

1. lib/validations/settings.ts 생성 (Research 참조):
   ```typescript
   import { z } from 'zod'

   export const metricsConfigSchema = z.object({
     view_target_min: z.number().int().min(0).max(10000),
     view_target_max: z.number().int().min(1).max(100000),
     like_target_min: z.number().int().min(0).max(1000),
     like_target_max: z.number().int().min(1).max(10000),
     ramp_days: z.number().int().min(1).max(30),
     curve_strength: z.number().min(0.1).max(2.0),
   }).refine(
     (data) => data.view_target_max > data.view_target_min,
     { message: '최대값은 최소값보다 커야 합니다', path: ['view_target_max'] }
   ).refine(
     (data) => data.like_target_max > data.like_target_min,
     { message: '최대값은 최소값보다 커야 합니다', path: ['like_target_max'] }
   )

   export type MetricsConfigInput = z.infer<typeof metricsConfigSchema>
   ```

2. app/actions/settings.ts 생성 (Research Pattern 4 참조):
   **updateMetricsConfig(formData: FormData)**
   - admin role 검증 (defense in depth)
   - FormData에서 각 필드 추출 후 Number 변환
   - Zod 검증
   - global_metrics_config 테이블 업데이트 (단일 행)
   - revalidatePath('/settings')
   - 에러 시 { error: { field: [messages] } } 반환
   - 성공 시 { success: true } 반환

모든 Supabase 쿼리에 `as any` 타입 assertion 사용
