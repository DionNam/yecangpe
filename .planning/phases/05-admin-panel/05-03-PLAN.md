# 05-03: User Management

## Goal

관리자가 구직자와 구인자 목록을 조회하고, 상세 정보를 확인하며, 계정을 비활성화할 수 있는 사용자 관리 기능을 구현한다.

## Requirements Covered

- ADMU-01: 관리자는 구직자 목록(이름, 이메일, 국적, 레벨, 직업, 가입일)을 볼 수 있다
- ADMU-02: 관리자는 구직자 상세 정보를 볼 수 있다
- ADMU-03: 관리자는 구인자 목록(이름, 이메일, 기업명, 가입일, 공고수)을 볼 수 있다
- ADMU-04: 관리자는 구인자 상세 정보와 해당 공고 목록을 볼 수 있다
- ADMU-05: 관리자는 사용자 계정을 비활성화할 수 있다

## Context

05-01에서 관리자 인증과 대시보드 레이아웃 완성됨. RLS에서 admin은 모든 users, seeker_profiles, employer_profiles 읽기 가능. users.is_active 필드로 계정 활성화 상태 관리. N+1 쿼리 방지를 위해 join 사용.

@.planning/phases/05-admin-panel/05-RESEARCH.md (Pitfall 5: User List Performance 참조)
@supabase/migrations/00001_create_base_schema.sql (users, seeker_profiles, employer_profiles 스키마)
@supabase/migrations/00002_create_rls_policies.sql (Admin can read all * 정책)

   const { data } = await (supabase as any)
     .from('users')
     .select(`
       id, email, is_active, created_at,
       employer_profiles (
         company_name
       )
     `)
     .eq('role', 'employer')
     .order('created_at', { ascending: false })

   // 각 employer별 공고수 조회
   const postsCount = await Promise.all(
     data.map(async (emp) => {
       const { count } = await supabase
         .from('job_posts')
         .select('*', { count: 'exact', head: true })
         .eq('author_id', emp.id)
       return { userId: emp.id, count: count || 0 }
     })
   )
