# 05-01: Admin Auth and Layout

## Goal

관리자 전용 인증 미들웨어와 사이드바 기반 대시보드 레이아웃을 구성하여 Phase 5 전체의 기반을 마련한다.

## Requirements Covered

- None (infrastructure - enables ADMP-*, ADMU-*, ADMM-*)

## Context

apps/admin은 기본 Next.js 스켈레톤으로 존재한다. 현재 middleware.ts는 세션만 관리하고 role 검증이 없다. Tailwind와 shadcn/ui가 설치되어 있지 않다. apps/web의 패턴(Tailwind v4, shadcn/ui)을 따라 설치하고, 관리자 role 체크를 middleware에 추가해야 한다.

**Database:** users.role = 'admin' 체크, is_admin() SECURITY DEFINER 함수 존재
**RLS:** Admin can read all users/posts/profiles per 00002_create_rls_policies.sql

2. postcss.config.mjs 생성 (TailwindCSS v4 방식 - apps/web과 동일):
   ```js
   const config = {
     plugins: {
       '@tailwindcss/postcss': {},
     },
   }
   export default config
   ```

3. app/globals.css 생성 (TailwindCSS v4):
   ```css
   @import 'tailwindcss';
   ```

4. lib/utils.ts 생성 (cn helper):
   ```typescript
   import { clsx, type ClassValue } from 'clsx'
   import { twMerge } from 'tailwind-merge'

   export function cn(...inputs: ClassValue[]) {
     return twMerge(clsx(inputs))
   }
   ```

5. shadcn/ui 초기화 및 컴포넌트 설치:
   ```bash
   pnpm dlx shadcn@latest init
   # 프롬프트에서: New York style, CSS variables, lib/utils.ts path
   pnpm dlx shadcn@latest add sidebar button badge card separator
   ```
