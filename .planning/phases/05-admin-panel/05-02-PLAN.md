# 05-02: Post Approval Workflow

## Goal

관리자가 심사중 공고를 승인/반려하고, 모든 공고를 수정하며, 직접 공고를 등록할 수 있는 공고 관리 기능을 구현한다.

## Requirements Covered

- ADMP-01: 관리자는 심사중 공고 목록을 볼 수 있다
- ADMP-02: 관리자는 공고를 승인하여 "게시됨" 상태로 전환할 수 있다
- ADMP-03: 관리자는 공고를 반려하며 반려 사유를 입력할 수 있다
- ADMP-04: 관리자는 모든 공고의 제목, 내용, 국적, 회사명을 수정할 수 있다
- ADMP-05: 관리자는 직접 공고를 등록하면 즉시 "게시됨" 상태가 된다

## Context

05-01에서 관리자 인증과 대시보드 레이아웃이 완성됨. RLS 정책에서 admin은 모든 job_posts 읽기/수정/삭제 가능. 승인 시 published_at 타임스탬프 설정 필수 (metrics 계산에 사용).

2. app/actions/posts.ts 생성 (Research Pattern 3 참조):

   **approvePost(postId: string)**
   - admin role 검증 (defense in depth)
   - job_posts.review_status = 'published'
   - job_posts.published_at = new Date().toISOString() (중요!)
   - job_posts.rejection_reason = null
   - revalidatePath('/posts', '/posts/pending')

   **rejectPost(postId: string, reason: string)**
   - admin role 검증
   - job_posts.review_status = 'rejected'
   - job_posts.rejection_reason = reason
   - revalidatePath('/posts', '/posts/pending')

   **updatePost(postId: string, formData: FormData)**
   - admin role 검증
   - title, content, company_name, target_nationality 업데이트
   - Zod 검증
   - revalidatePath('/posts', `/posts/${postId}`)

   **createAdminPost(formData: FormData)**
   - admin role 검증
   - global_metrics_config에서 view_target, like_target 범위 조회
   - 랜덤 target 값 생성
   - review_status = 'published', published_at = now()
   - author_id = admin user.id
   - revalidatePath('/posts')

모든 Supabase 쿼리에 `as any` 타입 assertion 사용 (established pattern)
</action>
